# Blindside

CNN adversarial attack generator. Implements FGSM and PGD from scratch in PyTorch, no attack libraries (no CleverHans, no torchattacks), targeting a pretrained ResNet20.

## What it does

### High-level concept: Model Bypassing via Imperceptible Perturbation

Blindside computes the gradient of the loss with respect to the input, and perturbs a correctly classified image just enough to flip the model's prediction while keeping the perturbation visually imperceptible.

<img width="1761" height="637" alt="image" src="https://github.com/user-attachments/assets/1dd6d85a-c3b6-4dfc-8529-846984a9a3fb" />


This *example* diagram shows a panda classified at 57.7% confidence that gets misclassified as a gibbon at 99.3% confidence after a small perturbation (epsilon = 0.007) in the direction of the gradient sign. Same image to a human. Garbage to the model.

Our current CIFAR-10/ResNet20 implementation generates a similar, much lower-resolution comparison grid (as shown in our results).

## Attacks implemented

**FGSM (Fast Gradient Sign Method)**
Single step. Perturbs the image in the direction of the sign of the gradient, scaled by epsilon.

```
x_adv = x + epsilon * sign(∇x J(θ, x, y))

```

**PGD (Projected Gradient Descent)**
Iterative version of FGSM. Takes multiple small steps, projecting back into the epsilon-ball after each step to keep the perturbation bounded.

## Project structure

The current project structure is a single-level directory with core files and data.

```
blindside/ (Project Root)
├── data/               # Download directory for the CIFAR-10 test dataset.
├── adversarial_batch.pt # Serialized attack data generated by evaluate.py.
├── attack.py           # Core attack implementations (FGSM and PGD mathematical methods).
├── evaluate.py         # Batch evaluation and attack generation entry point. Downloads dataset, loads ResNet20, and runs the evaluation loop.
├── visualise.py        # Visualizer script that loads adversarial_batch.pt and generates the result grid.
├── blindside_results.png # The final comparison grid generated by visualise.py.
├── pyrightconfig.json  # Pyright linter configuration.
└── README.md

```

## Usage

Running the project is a two-step process to generate evaluation data and then visualize it.

### Step 1: Generate Evaluation Data

Run the main evaluation script. This will download the CIFAR-10 dataset (if not present) and generate a batch of adversarial examples using FGSM, saving them to `adversarial_batch.pt` in the root directory.

`python evaluate.py`

*Note: The current ResNet20 model has zero clean accuracy on the CIFAR-10 test set due to a preprocessing mismatch. The generated adversarial batch will still be saved.*

<img width="727" height="893" alt="image" src="https://github.com/user-attachments/assets/cb6d9c0f-f613-40d6-a962-0a4f3b742328" />



### Step 2: Visualize Results

After Step 1 is complete, run the visualization script to load the generated data and generate a comparison grid showing clean images, magnified perturbation noise, and adversarial images. The final grid will be saved to `blindside_results.png`.

`python visualise.py`

*(The result will be a grid like `blindside_results.png`, which is saved in the root directory).*
